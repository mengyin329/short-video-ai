# 零消耗配置指南

## 概述

本 Agent 支持**完全零消耗**方案：所有功能（剧本创作 + 图像生成）都使用用户自己的 API 配置，服务提供方零成本运营。

## 用户配置流程

### 第一步：用户首次访问时

Agent 会自动引导用户配置自己的 API：

```
🎬 欢迎使用短视频AI编剧助手！

在使用之前，请先配置你自己的 API，这样才能使用所有功能：

📋 需要配置的内容：
├─ 1️⃣ 大模型 API（用于剧本创作）
│     • API Key
│     • Base URL
│     • 模型名称
└─ 2️⃣ 图像生成 API（用于分镜图/设定图，可选）
      • API Key
      • Base URL

💡 支持的服务商：
• OpenAI (GPT-4)
• 豆包/火山引擎
• DeepSeek
• Kimi/月之暗面
• 其他 OpenAI 兼容 API

请告诉我你的配置信息，格式如下：
---
大模型配置：
• API Key: sk-xxxx
• Base URL: https://api.xxx.com/v1
• 模型: gpt-4

图像生成配置（可选）：
• API Key: sk-xxxx
• Base URL: https://api.xxx.com/v1
---

配置完成后即可开始创作！🎉
```

### 第二步：用户提供配置

用户按格式提供配置信息后，Agent 确认配置生效。

---

## API 调用方式

### 方式一：通过 HTTP Header（推荐）

```bash
curl -X POST http://your-server:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "x-user-api-key: sk-xxxx" \
  -H "x-user-base-url: https://api.openai.com/v1" \
  -H "x-user-model: gpt-4" \
  -H "x-user-image-api-key: sk-xxxx" \
  -H "x-user-image-model: https://api.openai.com/v1" \
  -d '{
    "messages": [
      {"role": "user", "content": "帮我创作一个短视频剧本"}
    ]
  }'
```

### 方式二：OpenClaw 配置

在 OpenClaw 中配置时，需要在请求时动态传入用户配置：

```json
{
  "providers": [
    {
      "name": "screenplay-agent",
      "baseUrl": "http://your-server:8000/v1",
      "apiKey": "placeholder",
      "models": [
        {
          "id": "screenplay-agent",
          "name": "短视频编剧助手"
        }
      ]
    }
  ]
}
```

## 配置字段说明

### 大模型配置

| Header | 说明 | 必填 |
|--------|------|------|
| `x-user-api-key` | 大模型 API Key | ✅ |
| `x-user-base-url` | 大模型服务地址 | ✅ |
| `x-user-model` | 模型名称 | ✅ |
| `x-user-temperature` | 温度参数（0-2） | ❌ |

### 图像生成配置

| Header | 说明 | 必填 |
|--------|------|------|
| `x-user-image-api-key` | 图像生成 API Key | ❌（但生图功能需要） |
| `x-user-image-model` | 图像生成服务地址 | ❌（但生图功能需要） |

|------|-----------|
| 剧本创作 | ✅ 用户自己的大模型 API |
| 分镜图生成 | ✅ 用户自己的图像 API |
| 场景设定图 | ✅ 用户自己的图像 API |
| 角色设定图 | ✅ 用户自己的图像 API |
| **服务提供方** | **零消耗** ✅ |

---

## 注意事项

1. **API Key 安全**：建议使用 HTTPS 传输
2. **配置检测**：未配置图像 API 时，生图功能会提示用户配置
3. **并发限制**：用户自行控制 API 调用频率
4. **计费透明**：所有消耗都在用户自己的 API 账户
