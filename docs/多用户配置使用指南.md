# 多用户自定义配置使用指南

## 功能说明

Agent 支持多用户使用，每个用户可以使用自己的 API Key 和模型配置。

## 使用方式

通过 HTTP 请求头传递用户自定义配置：

| Header 名称 | 说明 | 必填 |
|------------|------|------|
| `x-user-api-key` | 用户的 API Key | 是 |
| `x-user-base-url` | 用户的模型服务地址 | 是 |
| `x-user-model` | 用户想使用的模型名称 | 是 |
| `x-user-temperature` | 温度参数（0-2） | 否，默认 0.7 |

## 调用示例

### 方式一：使用 /run 接口

```bash
curl -X POST http://localhost:8000/run \
  -H "Content-Type: application/json" \
  -H "x-user-api-key: your-api-key" \
  -H "x-user-base-url: https://api.openai.com/v1" \
  -H "x-user-model: gpt-4" \
  -H "x-user-temperature: 0.8" \
  -d '{"messages": [{"role": "user", "content": "你好"}]}'
```

### 方式二：使用 OpenAI 兼容接口

```bash
curl -X POST http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "x-user-api-key: your-api-key" \
  -H "x-user-base-url: https://api.openai.com/v1" \
  -H "x-user-model: gpt-4" \
  -d '{
    "messages": [
      {"role": "user", "content": "帮我创作一个职场短视频剧本"}
    ]
  }'
```

### 方式三：使用流式接口

```bash
curl -X POST http://localhost:8000/stream_run \
  -H "Content-Type: application/json" \
  -H "x-user-api-key: your-api-key" \
  -H "x-user-base-url: https://api.openai.com/v1" \
  -H "x-user-model: gpt-4" \
  -d '{"messages": [{"role": "user", "content": "你好"}]}'
```

## OpenClaw 配置示例

在 OpenClaw 的 `models.json` 中配置：

```json
{
  "providers": [
    {
      "name": "screenplay-agent",
      "baseUrl": "http://your-server:8000/v1",
      "apiKey": "placeholder",
      "models": [
        {
          "id": "screenplay-agent",
          "name": "短视频编剧助手"
        }
      ]
    }
  ]
}
```

**注意**：在 OpenClaw 中，需要在请求时动态传入用户配置。可以使用 OpenClaw 的自定义 Header 功能。

## 支持的模型服务

理论上支持所有 OpenAI 兼容的 API 服务：

| 服务商 | Base URL 示例 |
|--------|--------------|
| OpenAI | `https://api.openai.com/v1` |
| Azure OpenAI | `https://your-resource.openai.azure.com/openai/deployments/your-deployment` |
| 豆包/火山引擎 | `https://ark.cn-beijing.volces.com/api/v3` |
| DeepSeek | `https://api.deepseek.com/v1` |
| Kimi/月之暗面 | `https://api.moonshot.cn/v1` |
| 本地 Ollama | `http://localhost:11434/v1` |

## 工作原理

1. 用户发起请求，在 Header 中携带自己的配置
2. Agent 服务从 Header 中提取配置
3. 动态创建使用用户配置的 LLM 实例
4. 使用用户的 API 和模型执行请求
5. 返回结果

```
用户请求 → 提取 Header 配置 → 动态创建 LLM → 执行 → 返回结果
```

## 注意事项

1. **API Key 安全**：API Key 通过 HTTP Header 传输，建议使用 HTTPS
2. **配置优先级**：用户配置优先于系统默认配置
3. **工具调用**：用户自定义配置模式下，工具（分镜图、设定图生成）仍然使用系统配置
4. **并发限制**：每个请求会创建新的 LLM 实例，注意并发控制
