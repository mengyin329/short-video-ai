## Module A: 表演调度深化

本模块扩展核心版3.3节，提供更精细的表演拆解方法论。

### A.1 面部表情——肌肉组拆解法

将面部分为五个控制区域，每个区域独立描述：

```
┌──────────────┐
│ ① 额头/眉毛  │ 上扬=惊讶/疑问  内收=愤怒/专注  平展=平静
│ ② 眼部       │ 睁大=震惊/恐惧  微眯=怀疑/得意  半闭=疲惫/沉醉
│ ③ 鼻部       │ 鼻翼张开=愤怒/喘息  皱鼻=厌恶  正常=中性
│ ④ 嘴部       │ 上扬=喜悦  下拉=悲伤  抿紧=隐忍  张开=惊讶/呐喊
│ ⑤ 下颌/颈部  │ 紧咬=压力  颤抖=恐惧/哭泣  前伸=挑衅  后缩=退避
└──────────────┘
```

**混合表情处理流程**：
1. 识别混合情绪的成分（如"强忍泪水的微笑"= 悲伤 + 勉强的喜悦）
2. 将每种成分分配到不同面部区域
3. 描述区域之间的矛盾张力

示例——"强忍泪水的微笑"：
```
嘴角向上拉出弧度但肌肉僵硬不自然，
眼眶泛红蓄满泪水，睫毛微微颤动，
眉头中部轻微上挑形成悲伤纹路，
下颌肌肉紧绷控制住颤抖
```

### A.2 肢体姿态——三锚点法

每个姿态描述必须锚定三个参数：

**锚点①：重心**
- 重心在哪只脚 / 哪个支撑面
- 重心偏移方向暗示心理倾向（前倾=主动/攻击，后倾=被动/防御，侧倾=犹豫/闪避）

**锚点②：脊柱线**
- 挺直/前弯/后仰/侧弯/扭转
- 脊柱弧度直接表达角色当下的心理能量水平

**锚点③：手部状态**
- 手是"第二张脸"：手指的张合、力度、接触方式都传递信息
- 描述精度：指节级别
  - ❌ 握拳 → ✅ 五指缓慢蜷曲收紧，指甲陷入掌心，拇指压在食指第二关节上
  - ❌ 摊手 → ✅ 五指自然张开，掌心朝上，手指微微弯曲呈放松状态

### A.3 视线设计——控制观众注意力

**视线方向编码**：
```
        上
        ↑
  左上 ↗   ↖ 右上
左 ←    ●    → 右    （角色面向观众时的方向）
  左下 ↙   ↘ 右下
        ↓
        下
```
- 用时钟方位描述："视线指向两点钟方向"
- 或用画面参照："目光越过镜头左侧边缘，投向画面外"

**视线焦点状态**：
- 实焦：目光锁定具体对象，瞳孔聚焦，眼球稳定
- 虚焦/失焦：目光放空，瞳孔散大，不追踪任何物体
- 搜索：眼球快速小幅移动，在某个区域内寻找目标
- 闪避：视线短暂接触后快速移开

**视线的叙事功能**：
- 两个角色的视线交汇 → 对抗或连接
- 角色看向画面外 → 暗示画面外有信息，引导观众好奇心
- 角色直视镜头 → 打破第四面墙，制造压迫感或亲密感

### A.4 微动作库

| 微动作类型 | 情绪含义 | Prompt描述 |
|-----------|---------|-----------|
| 吞咽 | 紧张/恐惧/压制情绪 | 喉结上下滚动一次 |
| 屏息 | 震惊/专注/屏气凝神 | 胸腔停止起伏，身体完全静止 |
| 叹气 | 疲惫/释然/无奈 | 肩膀先微微上提，随后缓缓塌落，气息从鼻腔呼出 |
| 频繁眨眼 | 紧张/不安/撒谎 | 眼皮快速连续闭合三到四次 |
| 停止眨眼 | 震惊/出神/恐惧僵直 | 眼睛保持睁开不眨，目光定住 |
| 嘴唇抿动 | 犹豫/思考/控制发言 | 双唇轻轻向内抿合再松开 |
| 手指轻敲 | 焦躁/等待/思考 | 食指和中指交替轻叩桌面 |
| 无意识摸脸 | 不安/自我安抚 | 手指无意识地沿脸颊滑过 |
| 攥紧衣角 | 紧张/害怕/不安 | 手指蜷曲抓紧衣服下摆，指节泛白 |
| 身体微晃 | 犹豫/不稳/眩晕 | 身体轴线轻微左右摇摆 |

## Module B: 光影色彩转译

### B.1 情绪→光影 映射表

将核心版Phase 3中分镜表第5列（情绪&节奏）转化为光影Prompt的参考表。

| 情绪关键词 | 光影方案 | Prompt描述 |
|-----------|---------|-----------|
| 压抑/绝望 | 顶光，高对比，深重阴影 | 正顶光照射，面部形成浓重的眼窝阴影，背景几乎全黑 |
| 温暖/安宁 | 黄金时段侧光，柔和阴影 | 暖金色侧光从窗口洒入，阴影边缘柔和过渡 |
| 紧张/不安 | 不稳定光源，冷暖交织 | 忽明忽暗的闪烁光源，冷蓝与暖橙在面部交替 |
| 神秘/未知 | 逆光/剪影，大面积暗部 | 人物逆光呈剪影状态，仅边缘轮廓有一圈光晕 |
| 愤怒/爆发 | 强硬侧光，红橙暖色主导 | 强烈侧光切割面部，一半亮一半暗，暖红色调笼罩 |
| 悲伤/孤独 | 散射冷光，低饱和 | 阴天散射冷光，整体色彩低饱和偏灰蓝 |
| 希望/觉醒 | 暗中光束，暖调渐入 | 一束暖光从黑暗中穿透而来，照亮人物面部 |
| 恐惧/惊悚 | 底光，不完整照明 | 从下方打出的光源，面部阴影向上扭曲，背景漆黑 |
| 甜蜜/浪漫 | 柔焦暖光，微光粒子 | 柔和暖光笼罩，空气中飘浮着细微光点 |
| 决绝/壮烈 | 硬朗逆光，高光溢出 | 强烈逆光在人物周围形成燃烧般的光晕 |
| 日常/平和 | 自然光，中性色温 | 自然日光均匀照射，阴影柔和，色彩还原真实 |
| 癫狂/失控 | 多方向混乱光源，高饱和 | 来自多个方向的彩色光源交叉照射，色彩浓烈刺眼 |

### B.2 色彩心理参考

| 色调 | 心理暗示 | 适用场景 |
|------|---------|---------|
| 暖金/琥珀 | 怀旧、温情、回忆 | 回忆段落、亲情场景 |
| 冷蓝/青灰 | 疏离、理性、孤独 | 都市夜景、孤独角色 |
| 红/深红 | 危险、激情、暴力 | 冲突高潮、情欲场景 |
| 绿/墨绿 | 自然、诡异、腐朽 | 森林、实验室、病态场景 |
| 紫/靛 | 神秘、高贵、超自然 | 魔法场景、权力象征 |
| 橙/暖橙 | 活力、焦虑、紧迫 | 夕阳、爆炸前、追逐 |
| 低饱和/消色 | 绝望、虚无、死亡 | 战争场景、角色死亡 |
| 高饱和 | 梦境、幻觉、癫狂 | 超现实段落、毒品幻觉 |

### B.3 动态光影设计

部分镜头的光影在镜头进行过程中需要发生变化。在Prompt中描述光影变化的格式：

```
起始光影状态 → 触发点 → 终止光影状态
```

示例：
```
画面起始时房间昏暗仅有冷蓝色月光，角色推开窗帘，暖金色晨光涌入照亮面部
```

## Module D: 首帧Prompt工具适配

核心版输出的首帧参考图Prompt为通用中文格式。当用户使用不同生图工具时，按以下规则适配。

### D.1 通用格式（核心版默认）

```
[景别+视角]，[主体外貌与姿态]，[场景环境]，[光影氛围]，[色彩倾向]，[构图]，[全局后缀]
```
- 全中文，流畅的描述性语句
- 适用于可灵文生图、即梦文生图等中文原生平台

### D.2 可灵文生图 适配

- 直接使用通用格式，无需改动
- 可灵文生图对中文理解能力强，长句描述比堆砌关键词效果好
- 建议重点强调主体和光影，减少背景细节词量

### D.3 即梦文生图 适配

- 直接使用通用格式，无需改动
- 即梦对@引用的素材权重较高——如已有角色/场景参考图，优先使用@引用 + 简短描述
- 格式：`@角色标签 [姿态描述]，[场景描述]，[光影]，[风格后缀]`

### D.4 Midjourney 适配

当用户使用MJ生成首帧时，需将通用格式转化为MJ风格：

**转化规则**：
1. 改为英文
2. 将流畅描述句拆为逗号分隔的关键词短语
3. 添加MJ专属参数：`--ar 16:9 --v 7 --style raw` 等
4. 景别术语转为MJ理解的英文术语（close-up, wide shot, etc.）
5. 风格关键词前置

**转化示例**：
```
通用格式：
特写镜头，一位二十岁出头的东方女性面部，杏眼含泪，眉头微蹙，
嘴唇轻轻抿住，散落的黑色长发贴在脸颊，雨水顺着发丝滑落，
阴天散射冷光，低饱和偏灰蓝色调，电影质感浅景深

MJ格式：
cinematic close-up portrait, young East Asian woman, almond eyes glistening with tears, furrowed brows, pursed lips, long black hair clinging to wet cheeks, rain droplets on skin, overcast diffused cold light, desaturated blue-grey tones, shallow depth of field, film grain --ar 16:9 --v 7 --style raw
```

### D.5 Stable Diffusion / Flux 适配

当用户使用SD或Flux系列时：

**转化规则**：
1. 改为英文关键词组
2. 关键特征用括号加权：`(almond eyes:1.3)` `(rain on face:1.2)`
3. 画质词放在最前面：`masterpiece, best quality, ultra detailed`
4. 负面Prompt单独列出（使用全局负面Prompt的英文版）
5. 不使用MJ参数，改用采样器/步数等SD参数注释

**转化示例**：
```
正面Prompt：
masterpiece, best quality, ultra detailed, cinematic lighting,close-up portrait, (young East Asian woman:1.2), (almond eyes with tears:1.3), furrowed brows, pursed lips,long black hair clinging to cheeks, rain droplets, overcast cold light, desaturated blue-grey, shallow DOF, film grain

负面Prompt：
lowres, bad anatomy, bad hands, extra fingers, fewer fingers,cropped, worst quality, low quality, watermark, blurry
```

### D.6 工具自动识别

在首次Phase 0设定时，向用户确认首帧生成工具。之后每个镜头的首帧Prompt自动按对应工具格式输出。如用户在生产过程中更换工具，只需告知，后续自动切换。

如用户无明确偏好或未指定，默认输出通用中文格式。

## Module E: 平台深度优化

### E.1 可灵3.0 进阶技巧

**Elements角色锁定策略**：
- 每个角色上传3-5张不同角度的参考图到Elements
- 优先上传半身四分之三侧面（面部信息量最大且含身体信息）
- 参考图背景越干净，锁定越稳定
- 同一角色不同服装/状态作为不同Elements条目管理

**多分镜节奏控制**：
- 多分镜模式中，每个分镜的秒数分配影响节奏感
- 关键动作分镜可多给1秒，过渡分镜可压缩到2秒
- 示例：`镜头1（2秒）：环境空镜；镜头2（4秒）：角色特写，缓慢转头；镜头3（2秒）：反打对方震惊的表情`

**首尾帧模式的精确控制**：
- 当需要精确控制镜头的起止画面时使用
- 首帧图和尾帧图需保持视觉一致性（人物外观、场景、光线方向）
- 首尾帧之间的差异就是模型需要"补全"的运动
- 差异越小=动作越细腻；差异越大=动作越剧烈

### E.2 即梦 Seedance 2.0 进阶技巧

**四模态组合策略**：
- 图片定风格和构图：上传首帧参考图
- 视频定动作和运镜：上传动作参考短视频（如从影视片段中截取的动作参考）
- 音频定节奏和氛围：上传参考音乐或环境音
- 文字定细节和约束：Prompt补充图/视频/音频未涵盖的信息
- 最大组合：9张图 + 3段视频 + 3段音频 + 文字

**@引用权重管理**：
- 多个@引用同时使用时，排列顺序影响权重
- 优先写入最重要的引用（通常是角色 > 场景 > 动作）
- 文字Prompt与@引用冲突时，@引用权重通常更高——因此文字应补充而非重复@引用内容

**自分镜模式最佳实践**：
- 适用场景：空镜蒙太奇、情绪过渡段、氛围铺陈
- 用一段完整的叙事性文字描述，模型自动切分镜头
- 不适用场景：对话段落、精确表演、需要特定景别的镜头
- 可在自分镜叙事文字中嵌入节奏暗示："缓缓地...突然..."

### E.3 平台选择决策树

```
需要生成的镜头 →
├─ 有精确的首尾帧画面需求？→ 可灵（首尾帧模式）
├─ 有动作参考视频？→ 即梦（@视频引用）
├─ 有参考音频/需要音画节奏同步？→ 即梦（@音频引用）
```

## Module G: 高级分组策略

### G.1 正反打对话序列分组

对话场景的正反打镜头链处理规则：
- 将同一对话回合（如角色A说→角色B反应→角色B说→角色A反应）作为一个分组
- 组内镜头共享光影设定，保证切镜时光线一致
- 组内首镜头用图生视频建立视觉基调，后续镜头引用首镜头的风格

### G.2 蒙太奇序列分组

蒙太奇段落（时间流逝/情绪堆叠/平行叙事）的处理：
- 即梦平台优先：使用自分镜模式，将蒙太奇作为一段连续叙事文字投入
- 可灵平台：使用多分镜模式，但每个分镜描述需包含独立的场景信息
- 蒙太奇内部的镜头可以场景跳切，但需在分组说明中标注节奏模式

### G.3 一镜到底 / 长镜头处理

当分镜表中连续多个镜头实际属于同一个长镜头（无切）：
- 合并为单一Prompt，用时间锚点描述镜头内的变化
- 格式：`镜头从[起始构图]开始，[运镜描述1]，经过[角色/场景元素]，[运镜描述2]，最终停在[终止构图]`
- 时长可拉到模型支持的最大值

### G.4 闪回/梦境/异质空间处理

当分镜表中出现色调、画幅或画风与主时空不同的段落：
- 必须独立分组，不与主时空镜头合并
- 在该组的全局后缀中覆盖主时空后缀（如主时空是冷调写实，闪回改为暖调胶片+画面比例变化）
- 首镜头和末镜头需在生成注意中标注衔接方案（如渐隐渐显/闪白/色调过渡）

## Module H: 边界情况处理

### H.1 手部动作分级

| 风险等级 | 手部动作类型 | 处理策略 |
|---------|------------|---------|
| 低风险 | 自然下垂、叉腰、背手 | 正常写入Prompt |
| 中风险 | 持物（杯子/手机/武器）、指向 | 写入Prompt + 标注⚠️检查手指数量 |
| 高风险 | 双手交互（鼓掌/搓手/系扣子）| 考虑裁切画面避开手部，或用中景以上景别弱化手部 |
| 极高风险 | 精细操作（写字/弹琴/打字）| 建议改用特写只拍手部 + 后期合成，或规避该动作 |

### H.2 多角色同框

| 人数 | 处理策略 |
|------|---------|
| 2人 | 正常处理，主次分明，焦点角色描述在前 |
| 3人 | 需明确三角构图位置关系，描述谁在前景谁在中景谁在背景 |
| 4人以上 | 仅重点描述1-2个焦点角色，其余用"周围有数人"等模糊处理 |
| 群戏 | 焦点角色 + "背景是模糊的人群"，不逐一描述非重点角色 |

### H.3 特效镜头

| 特效类型 | Prompt处理策略 |
|---------|--------------|
| 爆炸/火焰 | 直接描述火焰形态、颜色、烟雾方向，AI模型对火焰有较好理解 |
| 雨/雪/雾 | 直接描述密度和方向，效果通常较好 |
| 魔法/能量 | 描述光效的形态、颜色、运动轨迹，避免过于抽象的描述 |
| 变形/扭曲 | 高风险——建议拆分为变化前和变化后两个镜头，中间过渡靠后期 |
| 快/慢动作 | 在Prompt中注明"慢动作"或"高速摄影效果" |

### H.4 文字/UI叠加镜头

如分镜要求画面中出现文字（如手机屏幕、信件内容、字幕卡）：
- AI生成的文字几乎不可控——不在Prompt中要求具体文字内容
- 策略：生成干净画面 + 后期叠加文字
- Prompt中可描述"手机屏幕亮着白光"而非"手机屏幕显示XXX消息"

## Module I: 质量检查清单

全部镜头Prompt输出完成后，逐项自检：

### I.1 Prompt质量检查

1. 每个Prompt的第一个信息是否是该镜头的视觉焦点？
2. 是否存在抽象情绪词？（必须为零）
3. 每个Prompt长度是否≤200字？
4. 是否存在否定句？（必须为零）
5. 信息优先级排序是否正确？

### I.2 视觉一致性检查

6. 角色描述是否与角色资产卡完全一致？
7. 同一场景内的光影是否连贯统一？
8. 全局后缀和负面Prompt是否正确附加？
9. 换装/时间变化是否在正确的镜头节点发生？

### I.3 表演质量检查

10. 表演调度是否具体到肌肉/关节级别？
11. 光影方案是否与分镜表的情绪列匹配？
12. 视线设计是否在对话镜头中逻辑正确（A看向B的位置）？

### I.4 平台适配检查

13. 可灵Prompt是否遵循其格式（文生视频/图生视频/多分镜）？
14. 即梦Prompt是否正确使用@引用？
15. 推荐平台的选择理由是否成立？

### I.5 风险标注检查

16. 手部复杂动作是否已标注⚠️？
17. 多角色交互是否已标注⚠️？
18. 镜头分组是否合理（可合并的未遗漏，不可合并的未强行合并）？

### I.6 音频标注检查

19. 所有对白镜头是否标注了具体台词内容？
20. 环境音/配乐指令是否与画面情绪一致？
